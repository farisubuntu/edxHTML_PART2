Responsive layout employs CSS and HTML to resize, hide, shrink, enlarge, or move the content so that it fits on any screen. Flexbox</span></h3> <p>When it comes to responsive web page design, you will want to create well&nbsp;structured&nbsp;HTML, and CSS layout to scaffold&nbsp;content. This is often referred to as a layout module, and it is helpful in managing complex responsive adjustment responsibilities.&nbsp;You can create a more traditional grid module which&nbsp;floats items together in rows or a much newer CSS technology called&nbsp;Flexbox.</p> <h4>Overview</h4> <p>Grid works well with the layout of items in two dimensions, such as rows <em>and</em> columns. Like every CSS property, or HTML element, there are defined ways to accomplish a task, but certain syntax and semantics are better suited for specific situations. In the Flexbox layout model, the children of a Flexbox container can be laid out in any direction, and can flex their sizes, either growing to fill unused space or shrinking to avoid overflowing the parent. You can easily manipulate both horizontal and vertical alignment of the children elements. You can build layouts in two dimensions by nesting the page elements (horizontal inside vertical or vertical inside horizontal). Essentially, if you have many items and don&rsquo;t need them to line up with any other elements on the page, but just want them displayed evenly and flexibly, you want to use Flexbox.</p> <blockquote> <p><strong>Note</strong>: Flexbox is newer to CSS and because of that browser support is limited. CSS Overview

Web designers use CSS to easily customize websites and control all aspects of presentation, while reducing the amount of bandwidth needed for a website. In addition, CSS requires that web designers only use one style sheet instructing the print parameters of a page instead of having to code each page individually. Style sheets describe how documents are presented on screens, in print, or perhaps how they are pronounced. By attaching style sheets to structured documents on the Web via HTML, web designers can influence the presentation of documents without sacrificing device-independence or adding new HTML tags.

The benefits of using CSS include:

    • Providing a consistent look and feel to a website
    • Using one or two stylesheets to maintain a website’s consistent colors, fonts, links, and other elements without having to hard code the same information on each and every page of the website
    • Saving time by quickly implementing global changes across a website via the style sheet
    • Creating websites that can respond to varying screen sizes
