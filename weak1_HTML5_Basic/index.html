Help

HeadingsMap is a browser extension which basic goal is being a tool that shows the headers structure of web documents so it helps auditing their accessibility.

The extension checks the document for extracting the headers and sections structure an shows the results in a hierarchical tree according to the hierarchy in the document. The results are shown in tabs, one for the headers structure and, if it is active, one for the sections structure (HTML 5 Outline).

Since its first version (released in May 5th, 2008), it had a lot of changes (mostly improvements but also bugfixes). One of them was the introduction of the HTML 5 outline algorithm for showing the sections structure of the documents. Sadly, according to what the W3C says ( HTML 5.2 Creating an outline), there are no browsers or assistive technology with native implementations of that algorithm, so it cannot be relied upon to convey document structure to users. Anyway, I decided to keep the analysis because the algorithm is still valid and I think that having it in a tool helps to the developers for adding a better structure to the documents.

Features

  • List of headings with information about their level and if they break the hierarchical structure
  • If there are elements that use role attribute with value heading, they are also listed and considered in the structure (their level is according to the value of aria-level attribute or 2 when it is not defined).
  • List of sections with header information and warnings for errors in the structure
  • Information from the main document and also from the documents inside iframes (includes a document selector that is active when there are multiple documents)
  • By clicking on the headers the document scrolls to the position of the header/section and highlights it (if the header is visible)
  • Detects changes in the DOM and when they finish it updates the trees
  • It can also be refreshed manually
  • Allow collapsing list of headers by levels
  • Dark theme (optional)
  • HTML 5 Outline test is optional
  • CSS outline for identifying the header (or section) when clicking the tree and if it is visible in the page
  • It doesn't show hidden headers or sections (hidden for assistive technologies)
  • If the header (or section) is an anchor, the link in the results has the href pointing to it (useful for being copied directly from the tree with the option in the contextual menu of the browser)
  • If header/section content uses aria-label, its value can be shown (optional property configurable in the settings section)
  • Version for multiple browsers (it uses the browserExtensions API):
    • Firefox
    • Chrome
    • Vivaldi (uses shame version as Chrome)

HeadingsMap tree - settings

Show heading level
It includes the heading level (1 to 6) that is used as tag (h1 to h6)
Identify errors
When active, it highlights the tree items that correspond to headers that don't follow the right order (for instance, one h4 after one h2 instead of being h3)
Error if not initial h1
It is commonly accepted that the first header in the document must be an h1. With this option active, and "identify errors" as well, the lack on this rule is considered error
Show hidden headers
When the document includes headers that are hidden for any tool (not only visually hidden, but also hidden for assistive technologies), they are not shown in the tree. When activating this option, they are also shown.

HTML 5 outline - settings

According to the W3C, there are not browsers or assistive technologies that support it, so having a good result in the test barely impact in the final accessibility. The HTML 5 algorithm is considered in the list of possible exclusions for the next Proposed Recommendation. If finally it is excluded from it, this feature will be removed from the extension.

Show section index
According to the algorithm, each section would have an index in the tree. This setting shows/hides that value
Show section tag
With this setting, the tree shows the HTML tag that generates the sections.
Identify errors
It highlights the tree items that have errors (sections without any header content)
Show hidden sections
When the document includes hidden sections (not only visually hidden, but also hidden for assistive technologies), they are not shown in the tree. When activating this option, they are shown. In long documents, this could affect to the performance since it requires to check all elements in the DOM
